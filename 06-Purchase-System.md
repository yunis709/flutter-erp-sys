# 📦 نظام المشتريات

## 🎯 مقدمة

نظام المشتريات يدير كامل دورة الشراء من طلبات الشراء إلى فواتير الموردين والمدفوعات، مع إدارة شاملة للموردين وتقييم أدائهم.

---

## 🏛️ هيكل النظام

```mermaid
flowchart TB
    subgraph Requests["الطلبات"]
        A[طلب شراء]
        B[موافقة الطلب]
        C[تحويل لأمر شراء]
    end
    
    subgraph Orders["الفواتير"]
        D[فاتورة شراء]
        E[استلام البضاعة]
        F[مطابقة الفاتورة]
    end
    
    subgraph Payments["المدفوعات"]
        G[دفع نقدي]
        H[دفع آجل]
        I[تقسيط الدفع]
    end
    
    subgraph Returns["المرتجعات"]
        J[مرتجع كامل]
        K[مرتجع جزئي]
        L[مرتجع تالف]
    end
    
    Requests --> Orders --> Payments
    Orders --> Returns
```

## 📋 طلبات الشراء

### سير العمل

```mermaid
flowchart LR
    A[تحديد الحاجة] --> B[إنشاء طلب]
    B --> C[مراجعة المخزون]
    C --> D[موافقة المدير]
    D --> E{موافق؟}
    E -->|نعم| F[تحويل لأمر شراء]
    E -->|لا| G[إلغاء/تعديل]
    F --> H[إرسال للمورد]
```

### حالات طلب الشراء

```mermaid
stateDiagram-v2

    معتمد --> محول: تحويل لأمر
    مرفوض --> مسودة: إعادة
    محول --> [*]
    مرفوض --> [*]
```

## 📄 فواتير الشراء

### سير العمل

```mermaid
sequenceDiagram
    participant S as المورد
    participant P as المشتريات
    participant Inv as المخزون
    participant Acc as المحاسبة
    
    S->>P: تسليم البضاعة + الفاتورة
    P->>Inv: استلام البضاعة
    Inv->>Inv: فحص الجودة
    Inv-->>P: تأكيد الاستلام
    
    P->>P: مطابقة الفاتورة
    P->>Acc: تسجيل الفاتورة
    
    alt فاتورة نقدية
        P->>Acc: دفع فوري
    else فاتورة آجلة
        Acc->>Acc: تسجيل ذمم
    end
    
    Acc->>Inv: تحديث المخزون
```

### القيود المحاسبية

#### فاتورة شراء آجلة

```
من حـ المخزون (مدين)                    1,000.00
من حـ ضريبة القيمة المضافة المدخلة (مدين) 150.00
    إلى حـ ذمم الموردين (دائن)                    1,150.00
```

#### دفع للمورد

```
من حـ ذمم الموردين (مدين)               1,150.00
    إلى حـ البنك/الصندوق (دائن)                   1,150.00
```

## 🔄 المرتجعات للموردين

### أنواع المرتجعات

```mermaid
mindmap

      منتج تالف
      غير مطابق
    لزيادة كمية
      طلب زائد
      تكرار
    لانتهاء الصلاحية
      منتهي
      قريب الانتهاء
    خطأ في الطلب
      منتج خاطئ
```

### سير المرتجع

```mermaid
flowchart LR
    A[تحديد المنتجات] --> B[سبب الإرجاع]
    B --> C[موافقة المدير]
    C --> D[إنشاء إذن إرجاع]
    D --> E[إرجاع للمورد]
    E --> F[استلام مبلغ/إشعار دائن]
```
