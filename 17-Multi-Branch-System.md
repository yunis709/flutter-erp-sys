# 🏪 نظام الفروع المتعددة

## 🎯 مقدمة

يقدم هذا المستند تصميم نظام الفروع المتعددة مع إدارة مركزية ومزامنة البيانات.

---

## 🏛️ هيكل الفروع

```mermaid
flowchart TB
    subgraph HQ["المركز الرئيسي"]
        MainDB[(قاعدة البيانات الرئيسية)]
        Sync[خدمة المزامنة]
        Reports[التقارير المركزية]
    end
    
    subgraph Branches["الفروع"]
        B1[فرع الرياض]
        B2[فرع جدة]
        B3[فرع الدمام]
    end
    
    subgraph Local["الخوادم المحلية"]
        L1[(DB فرعي)]
        L2[(DB فرعي)]
        L3[(DB فرعي)]
    end
    
    HQ --> B1
    HQ --> B2
    HQ --> B3
    
    B1 --> L1
    B2 --> L2
    B3 --> L3
    
    L1 -.->|مزامنة| MainDB
    L2 -.->|مزامنة| MainDB
    L3 -.->|مزامنة| MainDB
```

## 🔄 آلية المزامنة

### تدفق المزامنة

```mermaid
sequenceDiagram
    participant Branch as فرع
    participant Sync as خدمة المزامنة
    participant MainDB as قاعدة البيانات الرئيسية
    participant Queue as Message Queue
    
    Branch->>Sync: طلب مزامنة
    Sync->>Branch: جلب التغييرات المحلية
    
    Sync->>MainDB: تطبيق التغييرات
    MainDB-->>Sync: تأكيد
    
    Sync->>MainDB: جلب التحديثات المركزية
    MainDB-->>Sync: بيانات جديدة
    
    Sync->>Branch: تطبيق التحديثات
    Branch-->>Sync: تأكيد
    
    Sync->>Queue: نشر حدث المزامنة
```

### جدول المزامنة

| البيانات | التكرار | الاتجاه |
|----------|---------|---------|
| **الفواتير** | فوري | فرع ←→ مركز |
| **المخزون** | كل 5 دقائق | فرع ←→ مركز |
| **العملاء** | كل 15 دقيقة | فرع ←→ مركز |
| **التقارير** | كل ساعة | فرع → مركز |

        A1[بيانات الرياض]
    end

        A2[بيانات جدة]
