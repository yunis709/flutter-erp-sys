# 🧩 الوحدات النظامية

## 🎯 مقدمة

يحتوي نظام ERP على وحدات متكاملة تعمل معاً لتوفير حل شامل لإدارة المحلات التجارية.

---

## 🏛️ هيكل الوحدات

```mermaid
flowchart TB
    subgraph Core["النواة"]
        A[نظام المحاسبة]
        B[نظام المستخدمين]
        C[نظام الأمان]
    end
    
    subgraph Business["الأعمال"]
        D[نظام المبيعات]
        E[نظام المشتريات]
        F[نظام المخزون]
    end
    
    subgraph CRM["العلاقات"]
        G[نظام العملاء]
        H[نظام الموردين]
    end
    
    subgraph Analytics["التحليلات"]
        I[نظام التقارير]
        J[لوحة التحكم]
    end
    
    subgraph Support["الدعم"]
        K[نظام الإشعارات]
        L[نظام الفروع]
    end
    
    Core --> Business
    Business --> CRM
    Core --> Analytics
    Business --> Analytics
    CRM --> Analytics
    Support --> Business
    Support --> CRM
```

## 🔗 التكامل بين الوحدات

### تدفق البيانات

```mermaid
sequenceDiagram
    participant Sales as المبيعات
    participant Inv as المخزون
    participant Acc as المحاسبة
    participant Cust as العملاء
    participant Rep as التقارير
    
    Sales->>Inv: خصم من المخزون
    Sales->>Acc: تسجيل القيد
    Sales->>Cust: تحديث الرصيد
    Inv->>Rep: بيانات المخزون
    Acc->>Rep: بيانات مالية
    Cust->>Rep: بيانات العملاء
```

### القيود المحاسبية الآلية

| العملية | القيد المحاسبي |
|---------|----------------|
| **بيع نقدي** | من: الصندوق / إلى: المبيعات |
| **بيع آجل** | من: ذمم العملاء / إلى: المبيعات |
| **شراء نقدي** | من: المشتريات / إلى: الصندوق |
| **شراء آجل** | من: المشتريات / إلى: ذمم الموردين |
| **مرتجع مبيعات** | من: المبيعات المرتجعة / إلى: ذمم العملاء |
